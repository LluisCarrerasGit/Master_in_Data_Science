read_and_show_data <- function(name, col_names){
    path <- paste("gourmetdb/", name, sep = "", collapse = "")
    df <- read.csv(file=path, header=FALSE, sep=",", fileEncoding = "UTF-8")
    names(df) <- col_names
    print(paste("Visualitzem informaci del dataframe", name))
    cat("\n")
    print(head(df))
    cat("\n")
    print(dim(df))
    cat("\n")
    print(summary(df))
    return(df)
}

Claus foranes¶

Un cop carregades totes les dades, de tal manera que les dades de cada taula de la base de dades s'han carregat en un dataframe diferent, seguim mirant quina altra informaci cont el script SQL. El que ens trobem sn diferents comandes que especifiquen les claus foranes de les taules de la base de dades. Per exemle, aquesta s la primera comanda:

        ON UPDATE RESTRICT

El que ens diu aix s que la taula TIENDA t el seu atribut PAIS que s una clau forana que referencia l'atribut NOMBREPAIS de la taula PAIS. Si mirem la resta de comandes tenim la segent llista de claus foranes:

Regles d'asociació¶

Veiem que l'empresa del client a vegades sol fer promocions. Les promocions sn una bona eina per tal d'aconseguir ms ventes. Podriem ajudar al client dissenyant un sistema predictiu basat en regles d'asociaci. D'aquestes regles es pot treure un coneixement molt interessant. L'empresa del client podria emprar aquestes regles per detectar les comandes en les que hi "manquen" productes que es solen comprar junts amb els que hi ha a la comanda. Aix, les botigues poden preveure, amb una alta probabilitat, que en la prxima comanda el client inclour aquests productes que ara hi manquen, i aix les botigues ho poden aprofitar per oferir al client uns vals de descompte en aquests productes, que aix poden fer d'esquer per propiciar prximes comandes.

Per realitzar aquest sistema basat en regles d'asociaci ens calen les dades de les ventes que es fan, amb informaci de les comandes, els productes comprats i el client concret.
